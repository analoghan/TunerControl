Quick Start

  1. Load a MoTeC log file (.csv export or raw .ld) that contains knock level channels.
  2. Set the knock threshold (default 5%) — events above this level are classified as knock.
  3. Click Analyze.
  4. Review the per-cylinder distribution, RPM × Load grid, and timing recommendations.
  5. Use the RPM × Load grid (filtered by cylinder) to identify exactly where to pull timing in M1 Tune.

Input Requirements

Required Channels

  • Ignition Cylinder N Knock Level (1–8) — the per-cylinder knock intensity from the M1's DSP. At least one cylinder channel must be present.

Optional Channels (enhance analysis)

  • Engine Speed — RPM for binning and correlation
  • Inlet Manifold Pressure — MAP (kPa) for load binning
  • Ignition Timing — actual timing at the moment of knock
  • Ignition Timing Compensation — ECU's knock retard amount
  • Throttle Position — TPS for operating condition context
  • Gear — gear position for condition identification
  • Coolant Temperature — for thermal correlation
Tip: For the most useful analysis, ensure your log includes Engine Speed and Inlet Manifold Pressure at minimum. Without these, the RPM × Load grid and heatmap cannot be generated.

Knock Threshold

The threshold determines what constitutes a "knock event." Any sample where a cylinder's knock level exceeds this value (strictly greater than) is classified as knock.

  • Lower threshold (1–3%) — catches more events including borderline/light knock. Good for conservative tuning.
  • Default (5%) — reasonable starting point for most engines.
  • Higher threshold (8–15%) — only flags significant knock events. Use when you want to focus on heavy detonation only.
Note: The "correct" threshold depends on your knock sensor sensitivity, mounting location, and engine. Start at 5% and adjust based on whether the tool is flagging events that correlate with audible knock or timing retard from the ECU.

Report Sections

Diagnostic Summary

High-level overview: total events, most affected cylinder, worst operating conditions, and whether the ECU's knock retard is actively responding.

Per-Cylinder Knock Distribution

Table showing event count and percentage per cylinder. Identifies which cylinders are most knock-prone — useful for diagnosing injector issues, hot spots, or sensor problems.

Timing Adjustment Recommendations

Suggested timing reduction per cylinder based on knock frequency and severity. These are starting-point recommendations — always validate on the dyno.

Knock vs Engine Load (Normalized)

Shows knock rate (events/minute) per load bin, normalized by time spent at each load. Eliminates the bias of simply spending more time at certain loads. Sorted by rate so the most problematic load ranges appear first.

RPM × Load Grid (per Cylinder)

The most actionable output for tuning. This 2D table mirrors the layout of your MoTeC spark/ignition timing table:

  • Rows = Load (MAP in kPa), high to low (boost at top, vacuum at bottom)
  • Columns = RPM (500 RPM bins)
  • Cell values = knock events per minute, normalized by dwell time

How to use it

  1. Select a specific cylinder from the dropdown.
  2. Identify red cells — these are the RPM/Load zones where that cylinder is knocking.
  3. Open your ignition timing table in M1 Tune for that cylinder.
  4. Pull timing in the corresponding cells. Start with 1–2° and re-log.
  5. Repeat until the grid shows green (zero knock rate) across your operating range.

Display modes

  • Rate (events/min) — the primary metric. Accounts for how long you spent at each cell.
  • Event Count — raw number of knock events. Useful for seeing absolute volume.
  • Max Knock Level — worst-case intensity. High values here indicate severe detonation even if infrequent.
Tip: Grey cells (—) mean less than 0.5 seconds of dwell time — not enough data to draw conclusions. Drive more time at those RPM/Load points to fill in the map.

Charts

  • Per-Cylinder Knock Count — bar chart for quick visual comparison between cylinders
  • Knock Level vs RPM — scatter plot showing where in the RPM range knock occurs and how severe it is
  • Knock Heatmap (RPM × Load) — bubble chart showing knock concentration across the operating map
  • Ignition Timing vs RPM — scatter of timing values at knock events, colored by severity
  • Timing Retard Time Series — shows the ECU's knock retard response over time

Calculated Knock Frequencies

The primary knock frequency is determined by the cylinder bore diameter using the Draper formula for the first circumferential resonance mode of combustion chamber gas oscillation:

f = (α × c) / (π × B)

Where:

α = 1.841 (first circumferential mode Bessel function root)

c = speed of sound in combustion gases ≈ 900 m/s

B = cylinder bore in meters

ModeFrequencyUsage
1st circumferential (primary) 5,108 Hz Primary knock detection frequency
Below primary (−500 Hz) 4,600 Hz Lower bandwidth edge — catches temperature variation
Above primary (+500 Hz) 5,600 Hz Upper bandwidth edge — catches temperature variation
2nd harmonic 10,216 Hz Secondary detection — helps distinguish real knock from mechanical noise
Why ±500 Hz bandwidth? The actual knock frequency varies with combustion gas temperature. At higher loads and RPM, gas temperature increases, raising the speed of sound and shifting the knock frequency upward. The ±500 Hz window accounts for this variation across the operating range.

Knock Window Timing

The knock detection window defines when during the combustion cycle the M1 listens for knock. Setting this correctly is critical — too early catches compression noise, too late misses actual knock.

Start: 5° ATDC — just after TDC where peak cylinder pressure occurs

End: 60° ATDC — covers the expansion stroke where knock energy is released

Duration: ~55° of crank rotation

Tip: On the LT4 with boost, knock tends to occur earlier in the combustion cycle (closer to TDC) due to higher cylinder pressures. If you're missing events, try starting the window at 0° ATDC. If you're getting false triggers from piston slap, move the start to 8–10° ATDC.

Adaptive Threshold Setup

The M1's adaptive threshold tracks background noise and sets the knock detection level above it. Key parameters:

  • Noise Adaptation Rate — how quickly the background level adjusts. Slower = more stable but slower to adapt to changing conditions. Start at 0.5.
  • Threshold Multiplier — how far above background noise a signal must be to count as knock. Start at 2.0× (i.e., knock must be 2× louder than background).
  • Minimum Threshold — floor value to prevent false triggers at idle when background is very low. Set to 2–3%.

Knock Frequency Theory

Why Bore Size Determines Knock Frequency

Engine knock is the spontaneous ignition of the end-gas (unburned mixture ahead of the flame front). This creates a pressure wave that resonates within the cylindrical combustion chamber. The resonant frequency is determined by the chamber geometry — primarily the bore diameter.

The combustion chamber acts like a circular drum membrane. The dominant resonance mode is the first circumferential mode, where the pressure wave oscillates across the bore diameter.

Frequency Calculation for Any Engine

f₁ = (1.841 × c) / (π × B)

Where c ≈ 900 m/s and B = bore in meters

Simplified: f₁ ≈ 527,000 / B(mm)

Using the 2nd Harmonic for Noise Discrimination

Real knock produces energy at both the primary frequency and its harmonics (2×, 3×). Mechanical noise (valve train, piston slap, bearing noise) typically does NOT produce harmonic patterns. By monitoring both the primary (~5,100 Hz) and 2nd harmonic (~10,200 Hz), the M1 can distinguish real knock from false triggers:

  • Signal at primary AND 2nd harmonic → likely real knock
  • Signal at primary only → possibly mechanical noise
  • Signal at neither → no knock

Knock Tuning Workflow

  1. Baseline log: Run the engine through its full operating range with conservative timing. Log all knock channels at 1000 Hz minimum sample rate.
  2. Analyze: Load the log into this tool. Set threshold to 5% initially.
  3. Identify problem areas: Use the RPM × Load grid filtered by cylinder to find where knock is occurring.
  4. Pull timing: In M1 Tune, reduce ignition timing by 1–2° in the identified cells for the affected cylinder(s).
  5. Re-log and verify: Drive the same conditions again. Re-analyze. The knock rate should decrease.
  6. Iterate: Repeat until the grid shows zero or near-zero knock rate across your operating range.
  7. Advance cautiously: Once knock-free, you can try advancing timing 0.5° at a time in cells that show zero knock, looking for more power without triggering detonation.
Safety: Never advance timing beyond the point of knock on a supercharged engine like the LT4. Sustained detonation under boost can destroy pistons and bearings within seconds. Always have the M1's knock retard active as a safety net, even when manually tuning timing tables.
Pro tip: Log multiple sessions at different intake air temperatures and fuel qualities. Knock behavior changes significantly with ambient conditions. Your timing map should be safe under the worst conditions you expect to encounter (hot day, lower octane fuel).
